## Overview

v0.3.0 delivers polish and portability for the blog repository. First, fix the title duplication bug and add tags support to complete the publishing workflow. Then, add one-command bootstrap and dev container support so contributors can spin up the project without friction. Finally, adopt robust Python hook infrastructure and clean up accumulated technical debt. This milestone continues from v0.2.0 (phases 7-10) with phases 11-14.

## Milestone Summary

**Key Decisions:**
- Template defaults to draft: true for new posts
- Kepano fields use .optional().nullable() for schema flexibility
- Skills use blog: prefix matching GSD convention
- Node 22 major version only (auto patch updates)
- Named volume for node_modules (container performance)
- Python hooks use PEP 723 + uv pattern
- Logger writes to stderr and file for dual debugging
- trailingSlash: "ignore" for flexible routing
- Config consolidation: consts.ts is single source of truth

**Issues Resolved:**
- Title duplication in Obsidian template
- Skills not discoverable (wrong directory structure)
- SessionStart hook output not visible to users
- No timeout protection on hooks
- Dead code accumulation (16 files)
- Scattered configuration (constants.ts, consts.ts, config.ts)

**Issues Deferred:**
- Newsletter service setup (v0.4.0+)
- About page bio content (content work)
- `just doctor` health check (v0.4.0+)

**Technical Debt Incurred:**
- None. Milestone focused on reducing debt.
